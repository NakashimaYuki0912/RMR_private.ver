RMR GUI 美观化方案推荐 可部署到 Unity/RMR

三套完整视觉体系（Hub + 玩法介绍 + 图鉴）均只使用 RMR 已能实现的能力： Image / TMP / Button / ScrollRect / Mask / Overlay Canvas、纯色与现有 ArtWorks 贴图、 ApplyTmpFontPreservingSharpMaterial（禁止 CJK 伪粗体）。 不依赖 Web-only 特效。选定方案后，可在 RMR_StartHubPanel / RMR_HelpHandbookPanel / LogAtlasPanel 中落地。

部署约束

  • 目标分辨率基准 1920×1080，ScaleWithScreenSize
  • 中文必须 NotoSansCJKsc + 锐利材质路径
  • 交互与门控（存档/解放/重置）不得因换皮改坏
  • 优先复用 MysteryButton / 边框类 ArtWorks，缺则几何色块兜底

推荐选型原则

  • 与废墟图书馆：黑金、克制、仪式感、少饱和色
  • 信息密度：图鉴 > 玩法介绍 > Hub
  • 实现成本：几何色块 < 现有贴图拼装 < 新美术资源
  • 破坏性操作仅用暗红，不用大红高饱和

当前代码锚点

  • Hub：RMR_StartHubPanel.cs
  • 玩法介绍：RMR_HelpHandbookPanel.cs
  • 图鉴：LogAtlasPanel.cs
  • 字体：LogLikeMod.ApplyTmpFontPreservingSharpMaterial

方案 A · 接待邀请索引（Invitation Index）

延续 ADR-001 方向并做精修：全屏黑场 + 左侧身份/印记 + 右侧编号接待菜单。 玩法介绍为「章节索引 + 顶栏插图 + 连续滚动」。图鉴为「左分类轨 + 中央收藏墙 + 右详情」。 最贴近原版接待/邀请仪式感，推荐默认。
实现难度：低–中 复用现有票券按钮结构 与当前 WIP Hub 最接近 图鉴信息密度高

方案 B · 司书手稿 / 羊皮书（Manuscript）

居中「书页/邀请函」面板：Hub 像一封被打开的邀请信；玩法介绍像对开手稿（左目录右正文）；图鉴像书架横排，选中书脊高亮。 更安静、更「图书馆档案」——适合希望 UI 沉稳、减少左右分屏信息量的玩家。
实现难度：中 Hub 改回居中主面板 图鉴需改网格为横排书架 图鉴密度略降，分页会更多

方案 C · 十层印记 / 司书纹章（Sephirah Seal）

部署可行性结论

  • A / B / C 均可部署到当前 RMR：不引入 WebView、不依赖新 DLL 框架。
  • A 成本最低：与现有票券 Hub + 三栏图鉴代码同构，主要 polish。
  • B 需改 Hub 为居中面板 + 图鉴布局重写，约 1 次完整 UI 重构迭代。
  • C 中等：Hub/Help 结构可改，图鉴顶栏 pill 改动面中等；印记比例需实机微调。
  • 任选一案后：改 C# 构建 UI → deploy_workshop.ps1 → 完全重启游戏验收。
