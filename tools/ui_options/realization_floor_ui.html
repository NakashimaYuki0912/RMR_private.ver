方案 A

邀请书式纵向菜单

全屏墨黑背景，中央竖向「邀请书」卡片。十层做成单列长按钮（主按钮金边），信息密度低，仪式感接近原版发送邀请 / 开局 Hub。

视觉特征

单列列表 Mystery 金边按钮感 窄竖卡 仪式感最强
  • 与开局模式选择页同族，产品感统一
  • 中文大字最易读，不抢戏
  • 已通关仅用金色字/边区分

方案 B

司书层色块棋盘

仿图书馆选层：5×2 方格棋盘，每格用司书色侧条标识身份。已通关金框点亮。更像「图书馆地图」而不是菜单列表。

视觉特征

5×2 网格 司书色编码 宽横卡 最像原版楼层
  • 十层同屏，扫视最快
  • 颜色即楼层身份（历史绿黄、语言红…）
  • 已通关/未通关对比非常鲜明

方案 C · 当前代码方向

通行券 / 双列邀请卡

双列「通行券」：左侧罗马数字票根 + 虚线撕口 + 司书色轨，中间楼层名与英文码，右侧状态印章。顶部解放进度条。信息最全，偏精致 UI。

视觉特征

双列卡片 票根 + 印章 进度条 X/10 信息最全
  • 进度、状态、楼层身份同时可见
  • 已实现雏形在 LogRealizationPanel
  • 适合当「清层图鉴 + 入口」

方案 D

故事书页 / 诗句目录

更文学、更「废墟」：像书页目录。上方氛围引文，下列章节式楼层名 + 状态。装饰线、菱形饰，少色块，偏沉静叙事。

视觉特征

目录列表 引文氛围 少色块 氛围最强
  • 最有叙事感与文学味
  • 视觉安静，不抢原版美术
  • 适合作为重大挑战入口仪式页
